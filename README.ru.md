## 🚀 История обновлений

<details>
<summary><b>📦 v2.0.2 — Фикс багов (Текущая версия)</b></summary>

* **[Storage]** Полная переработка модели многопоточности в `jh_storage_manager.py`. Два независимых, никогда не вложенных лока заменяют исходный единый `_file_lock`: `_file_lock` (удерживается только во время `_modify_file()` — полный цикл чтения, изменения и записи файла) и `_url_lock` (удерживается микросекунды, только мутации in-memory `set`-объектов). Проверка дублей теперь — O(1) по всегда актуальным `_approved_urls` / `_rejected_urls`, заполняемым при старте, без единого обращения к диску.
* **[Storage]** Удалено поле `description` (сырой `document.body.innerText`, до нескольких МБ на вакансию) из одобренных записей. ИИ-анализ работает с payload'ом запроса в памяти — поле никогда не читалось обратно из файла. По мере накопления записей раздувшийся файл удерживал `_file_lock` дольше 8 секунд, вызывая таймауты всех параллельных webhook-запросов.
* **[Storage]** Однократная миграция при старте (`_migrate_strip_description()`) удаляет устаревшее поле `description` из всех существующих записей до запуска Flask или воркера. Без неё фикс хранилища помогал бы только новым сессиям.
* **[Enqueue]** Удалены `_enqueue_lock` и `_in_flight_urls` из `enqueue_vacancy()`. Прежняя тройная проверка дублей внутри лока сериализовала все 24 параллельных Flask-потока — при быстрых кликах проходила одна вакансия в секунду. Заменено: одна O(1)-проверка по хранилищу, затем безусловный `queue.put()`. Воркер-поток выполняет лёгкую дедуп-проверку для редких гонок при одновременных запросах.
* **[Startup]** Исправлена гонка при запуске Flask: UI ранее выставлял `is_active = True` до того, как Flask успевал привязаться к порту 5000. Клики расширения в этот момент возвращали `ERR_CONNECTION_REFUSED`. Исправлено через `threading.Event` (`_flask_ready`): Flask сигналит event сразу после успешного `make_server()`, UI опрашивает через `self.after()` без I/O на главном потоке. Кнопка показывает «ЗАПУСК…» и задизейблена до готовности сервера.
* **[Extension]** Убрана проверка `tab.status === 'loading'`. Современные SPA (LinkedIn, HH.ru, Greenhouse) никогда не выставляют `status = 'complete'` из-за фоновых запросов — проверка блокировала сбор вакансий на полностью загруженных страницах.
* **[Extension]** Текст страницы усекается до 50 000 символов перед отправкой. Полный `document.body.innerText` может достигать нескольких МБ; при запущенном OBS или высокой нагрузке 24 Flask-потока конкурировали за GIL во время `json.loads()` — последний запрос в пачке ждал 5+ секунд и выбивался по таймауту 8 с.

</details>
